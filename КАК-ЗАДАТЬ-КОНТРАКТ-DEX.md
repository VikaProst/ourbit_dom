# DEX / on-chain линия (фиолетовая [CA]) — как задать правильный контракт

Панель **🕵 MEXC↔DEX** берёт on-chain цену монеты автоматически через Dexscreener:
по тикеру находит самый ликвидный пул. Обычно это правильно, НО у популярных
тикеров бывают **коллизии** (несколько токенов с одним именем) — тогда фиолетовая
линия покажет цену не того токена.

## Как поправить вручную

Рядом с `server.py` есть файл `dex_map.json` (создаётся автоматически). В нём
кэш «тикер → контракт». Чтобы задать нужный токен жёстко — впиши строку вида:

```json
{
  "LINK": { "chain": "ethereum", "pair": "0xa6cc3c2531fdaa6ae1a3ca84c2855806728693e8" },
  "PEPE": { "chain": "ethereum", "pair": "0x11950d141ecb863f01007add7d1a342041227b58" }
}
```

- `chain` — сеть Dexscreener: `ethereum`, `bsc`, `base`, `solana`, `polygon`, `arbitrum`…
- `pair` — адрес **пары** (pool), не токена. Проще всего: открой монету на
  dexscreener.com, скопируй адрес из URL (`dexscreener.com/<chain>/<pair>`).

После правки — перезапусти `start.bat`. Ручная запись всегда побеждает авто-поиск.

## Отключить on-chain
Просто выключи чип **DEX·CA** в верхней панели окна — линия пропадёт, запросы к
Dexscreener прекратятся.

## DexTools вместо Dexscreener
У DexTools API платный ключ. Dexscreener бесплатный и даёт ту же USD-цену с самого
ликвидного пула. Если появится ключ DexTools — скажи, подключим его источником.
