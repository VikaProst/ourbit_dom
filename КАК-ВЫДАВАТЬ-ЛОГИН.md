# Как выдавать логины (для Вики) — ПРОСТОЙ режим, без сервера

Вход в терминал проверяется **локально** по файлу **`users.txt`** — никакого сервера и API.
В файле по одной паре на строку: `логин:пароль`.

> Вход закрывает **весь терминал**: без верного логина/пароля терминал не открывается.

---

## Где логины

Файл `users.txt` в папке `ourbit_dom`. Открой его Блокнотом. Внутри:
```
# логин:пароль — по одной паре на строку
vika:fd0a7b23
```

## Добавить логин другу

Допиши новую строку `логин:пароль` и сохрани (Ctrl+S). Например:
```
vika:fd0a7b23
vasya:1234
lena:qwerty7
```
Пару `vasya / 1234` отдаёшь другу — он вводит её на экране входа.

## Сменить пароль / отозвать доступ

- Сменить: поправь пароль в его строке, сохрани.
- Отозвать: удали его строку, сохрани. Вход перестанет работать (после перезапуска терминала).

> Изменения `users.txt` применяются при следующем **входе** (перезапускать сервер не обязательно —
> файл читается каждый раз при попытке входа).

---

## Как отдать другу его логин

У каждого друга — **свой** `users.txt` в его копии. Когда собираешь пакет другу
(`СОБРАТЬ-ПАКЕТ-ДРУГУ.bat`), положи в его папку `users.txt` с ЕГО строкой (одной парой),
а свою строку `vika:...` туда не клади.

- `users.txt` — личный файл, в GitHub не уходит (в `.gitignore`) и не затирается авто-обновлением.
- Можно и один общий пароль на всех — просто одна строка в `users.txt`. Но «каждому свой» надёжнее
  (сможешь отозвать одного, не трогая других).

---

## Если что-то не работает

- «неверный логин или пароль» → проверь строку в `users.txt`: формат `логин:пароль`, без лишних
  пробелов, двоеточие одно.
- Терминал открывается вообще без запроса логина → значит `users.txt` пустой или в нём только
  строки с `#`. Добавь хотя бы одну пару.
