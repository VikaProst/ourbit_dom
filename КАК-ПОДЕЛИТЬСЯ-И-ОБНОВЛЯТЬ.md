# Как поделиться с друзьями + авто-обновление (для Вики)

Цель: друзья ставят терминал ОДИН раз, а дальше твои фиксы прилетают им САМИ при запуске.
Механизм уже встроен: `start.bat` перед запуском тянет свежие файлы с одного адреса (GitHub).

---

## ЧАСТЬ 1 — настроить хранилище (один раз, ~15 минут)

### 1. Аккаунт и программа
1. Заведи аккаунт на https://github.com (если нет) — бесплатно.
2. Скачай и поставь **GitHub Desktop**: https://desktop.github.com (это GUI, командная строка НЕ нужна).
3. В GitHub Desktop войди своим аккаунтом.

### 2. Создать репозиторий из папки терминала
1. GitHub Desktop → **File → Add local repository** → выбери папку `ourbit_dom`.
2. Он скажет «это не репозиторий» → нажми **create a repository**.
3. Имя: `squad-terminal`. Галочку «Keep this code private» **СНИМИ** (публичный — чтобы друзья скачивали без пароля). → **Create**.
4. Нажми **Publish repository** (вверху) → снова сними приватность → **Publish**.

Теперь твой код на GitHub по адресу: `https://github.com/ТВОЙ_ЛОГИН/squad-terminal`

### 3. Прописать адрес обновления
1. Узнай «raw»-адрес: он такой →
   `https://raw.githubusercontent.com/ТВОЙ_ЛОГИН/squad-terminal/main`
   (замени ТВОЙ_ЛОГИН; ветка обычно `main`, иногда `master` — посмотри в GitHub Desktop слева сверху).
2. В папке `ourbit_dom` открой файл **`update_url.txt`** (если нет — создай) и впиши туда ЭТОТ адрес одной строкой. Сохрани.
3. Сгенерь список файлов: двойной клик по **`publish.py`** (или запусти его) — создаст `manifest.json`.
4. В GitHub Desktop внизу слева напиши в «Summary» что-нибудь («первая версия») → **Commit to main** → вверху **Push origin**.

Готово. Хранилище настроено.

---

## ЧАСТЬ 2 — что отдать друзьям (один раз каждому)

1. Заархивируй папку `ourbit_dom` в zip (правый клик → «Отправить → Сжатая папка»).
   Внутри уже есть `update_url.txt` с твоим адресом — друзьям ничего прописывать не надо.
2. Пришли им zip + скажи открыть **`ИНСТРУКЦИЯ-ДЛЯ-ДРУЗЕЙ.md`**.
   Там всё: поставить Python, запустить `start.bat`, загрузить расширение в Chrome, подключиться.

С этого момента при каждом запуске `start.bat` у них сам скачивает свежие файлы с твоего GitHub.

---

## ЧАСТЬ 3 — как выкатывать ФИКСЫ (каждый раз, ~1 минута)

Когда я (ассистент) поправлю баг у тебя локально:

1. Двойной клик по **`publish.py`** → пересоберёт `manifest.json` (хеши изменённых файлов).
2. GitHub Desktop → «Summary»: коротко что пофиксили → **Commit to main** → **Push origin**.

Всё. У друзей при следующем запуске `start.bat` фикс подтянется автоматически
(GitHub раздаёт файлы с задержкой до ~5 минут — если друг только что запустил, пусть перезапустит через пару минут).

## Частые вопросы

- **«Друг уже запущен, а я выкатила фикс»** — фикс применится при следующем запуске `start.bat`
  (обновление происходит ДО старта, не на лету — чтобы не сломать торговлю в процессе).
- **«Не хочу публичный репозиторий»** — можно приватный, но тогда друзьям нужен токен GitHub в адресе
  (сложнее). Для терминала секретов нет (токен биржи у каждого свой, из браузера), так что публичный безопасен.
- **«Обновились не те файлы / что-то сломалось»** — updater проверяет sha256 каждого файла и качает только
  совпавшие; личные файлы друга (`proxies.json`, `update_url.txt`) не трогает.
- **Личные настройки друга** (лоты, тема, раскладка) хранятся в его браузере (localStorage) — обновление их не сбрасывает.
