# Как выдавать/отзывать ключи (для Вики)

Ключи проверяет твоя «проверялка» на премиум-сервере (activation_server.py, порт 8790).
Правило: **1 ключ = 1 IP** — ключ намертво привязывается к первому компьютеру, где его включили.

Все команды запускать в папке `ourbit_dom` (например через PowerShell или двойным кликом по .bat-обёрткам ниже).

---

## Выдать ключ другу

```
python make_key.py Имя_друга
```
(например `python make_key.py Вася`)

Выведет **КЛЮЧ** — его и отдаёшь другу. Друг вписывает ключ в свой `license.txt` (см. его инструкцию).
Ключ сразу активен на сервере, ждать ничего не надо. Запишется в `keys_log.txt` (кому какой ключ).

---

## Посмотреть, кто активирован

```
python make_key.py list
```
Покажет сколько ключей и к каким IP они привязаны.

---

## Отозвать доступ (заблокировать друга)

1. Открой `keys_log.txt` — найди строку друга, скопируй его **хеш**.
2. ```
   python make_key.py del ХЕШ
   ```
Ключ перестанет работать сразу (и IP-привязка снимется).

---

## Сбросить привязку (друг сменил интернет/IP)

Если у друга поменялся IP и он не может активироваться — просто отзови старый ключ и выдай новый:
```
python make_key.py del ХЕШ_СТАРОГО
python make_key.py Имя_друга
```

---

## Как собрать папку для друга (важно!)

Перед тем как заархивировать `ourbit_dom` для друга:
1. Открой `license.txt` и **очисти его** (удали свой ключ) — чтобы не отдать другу свой.
   (Даже если забудешь — твой ключ у друга не сработает, он привязан к твоему IP. Но лучше очистить.)
2. Заархивируй папку в zip → пришли другу + скажи открыть `ИНСТРУКЦИЯ-ДЛЯ-ДРУЗЕЙ.md`.

Файлы `admin_secret.txt`, `act_keys.json`, `keys_log.txt` — **твои личные**, они НЕ попадают в GitHub
(в .gitignore). Но в zip папки они попадут — поэтому лучше слать другу zip БЕЗ них, либо не переживать:
`admin_secret.txt` другу бесполезен без доступа к серверу, но всё же не разбрасывайся им.

> Проще: держи отдельную «чистую» копию папки для раздачи друзьям (без license.txt/admin_secret.txt),
> а рабочую — у себя. Скажи ассистенту «сделай чистый пакет для друзей» — соберу автоматически.

---

## Если что-то не работает

- Друг пишет «🔑 АКТИВАЦИЯ: ключ недействителен» → проверь `python make_key.py list` (есть ли его ключ),
  и что он вписал ключ в `license.txt` без лишних пробелов.
- «сервер активации недоступен» → проверялка на сервере упала. Проверь: открой в браузере
  `http://45.153.247.114:8790/health` — должно быть `{"ok": true...}`. Если нет — перезапусти
  (скажи ассистенту, поможет) или `РАЗВЕРНУТЬ-ПРОВЕРЯЛКУ.bat` заново.
